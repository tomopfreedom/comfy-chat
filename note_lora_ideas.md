# LoRA 管理・学習 機能アイデアメモ

作成日: 2026-05-17  
状態: 未検討（アイデア段階）

---

## 現状の課題

- `loras.json` はフラットなJSON配列。LoRA数が増えると「探す・比べる・グループ管理する」がつらくなる
- `costume_map` は `description` 内テキストパース（`token=X, tags=Y,Z`）で、増えると編集ミスが起きやすい
- 学習は完全に別ツール任せで、「生成→評価→改善」サイクルがアプリ内で完結しない

---

## A. LoRA 管理 UI の改善

### A-1. 検索・フィルタ・ソート ★すぐできる
- 名前/説明のテキスト検索（インクリメンタルサーチ）
- base_model 別フィルタは既存。追加で「未使用」「衣装あり」などの絞り込み
- ソート：名前順 / 使用頻度順 / 追加日時順

### A-2. 使用頻度の記録 ★すぐできる
- 生成のたびに使ったLoRAのカウントをインクリメント
- `loras.json` に `use_count` フィールドを追加、またはサイドカーファイルで管理
- 「よく使う」ピン留め機能

### A-3. costume_map のテーブルUI ★中期
- 現在は `description` 内にテキスト埋め込み形式（パース任せ）
- 衣装パターンをテーブル行として視覚的に追加/削除/編集できるUI
- バックエンドのパースロジックは既存流用

### A-4. プレビューサムネイル ★すぐできる
- Civitai から取得した代表画像を `static/lora-thumbs/` に保存してグリッド表示
- 管理画面がリスト→カード型グリッドに変わるだけで大幅に見やすくなる

### A-5. グループ/タグ付け ★中期
- `loras.json` に `tags: ["キャラクター", "ウマ娘"]` フィールドを追加
- タグクリックでフィルタリング。既存の `base_model` フィルタと組み合わせ
- **スキーマ変更注意**: 既存 loras.json のマイグレーションが必要

### A-6. 未登録 LoRA の自動検出 ★中期
- ComfyUI の `/api/models/loras` レスポンスと `loras.json` の差分を表示
- 未登録ファイルをハイライトして「簡易登録」ボタンを出す
- FEATURES.md の候補 #2 と同一機能

## B. LoRA 学習パイプライン

RTX 5060 Ti（16GB VRAM）はキャラクターLoRA学習に十分なスペック。

### B-1. データセット管理 ★大型・先行実装推奨
- 学習用画像のアップロード/管理（`~/infra/lora-datasets/<lora-name>/images/` 等）
- 自動キャプション生成：WD14 Tagger（既存 `tags.json` 活用）または Vision LLM（既実装）と連携
- キャプションの手動編集（画像と横並び表示）

### B-2. 学習設定 UI → kohya-ss 連携 ★大型
- 学習バックエンド: `kohya-ss/sd-scripts`（最も実績あり）
- comfy-chat は `config.toml` を自動生成 → `python train_network.py` をサブプロセス実行する構造
- UI 項目候補:
  - ベースモデル選択
  - 解像度（512 / 768 / 1024）
  - エポック数 / ステップ数 / 学習率
  - 初心者向けプリセット（キャラクター用/スタイル用）

### B-3. 学習進捗モニタリング ★大型
- kohya-ss のログファイルを tail して WebSocket でブラウザに流す
- ロス値グラフ（canvas/SVG で描画、外部ライブラリ不要）
- 一定エポックごとに comfy-chat でサンプル画像を生成して表示

### B-4. 学習完了後の自動登録 ★大型
- 完了で `*.safetensors` を自動で `~/infra/comfyui/models/loras/` にコピー
- `loras.json` への仮登録エントリを自動生成（trigger_words は学習設定から引き継ぎ）

### B-5. AIレビューとの連携（comfy-chat 固有の強み）★大型
- 学習済みLoRA のサンプル生成 → Vision LLM AIレビュー（既実装）で品質スコアリング
- 複数 checkpoint を比較生成して「どのエポックが最良か」を評価
